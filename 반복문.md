## 반복문의 차이점

- **`for` 문:** 반복 횟수가 명확할 때 주로 사용합니다.
    - `for(초기화; 조건식; 증감식) { ... }`
    
- **`while` 문:** 조건이 참인 동안 무한히 반복합니다. (횟수보다 '상태'가 중요할 때)
    - `while(조건) { ... }`
    
- **`do...while` 문:** 조건이 거짓이더라도 **최소 1번은 무조건 실행**하고 싶을 때 사용합니다.

## 반복문 제어  break, countinue

반복문의 흐름을 강제로 바꾸는 키워드입니다.

- **`break`:** 만나자마자 해당 반복문을 **즉시 종료**하고 빠져나갑니다.
- **`continue`:** 현재 회차를 **건너뛰고** 다음 회차(증감식)로 바로 넘어갑니다.

### if문과 for문 - 특정 조건일때 코드 수행
```js
for(let i=0; i<arr.length;i++) {

 *         if(조건) {코드수행} 
   }
```

### if문과 break, continue 
```js
for(let i=0; i<arr.length;i++) {

       if(조건) {코드수행; break;} 
}

for(let i=0; i<arr.length;i++) {
	if(조건) {continue;} 
}
```

## 레이블문
반복문에대해서 참조할수있는 식별자를 제공합니다.

주로 중첩루프에서 어떤 루프를 break,continue할지 결정할 때 사용할수있습니다.

```js
outerLoop:
 *      while(조건) { 
	   코드수행
	 *        innerLoop: while(조건) {
		 *          if(특정조건이면) {
			 *            break markLoop;}
		 *          else if(다른조건이면){
		 *            break innerLoop;
   }}}
```

```js
outer: for (let i = 0; i < 3; i++) {
  for (let j = 0; j < 3; j++) {
    if (i + j === 3) break outer; // inner가 아닌 outer를 종료함
    console.log(i, j);
  }
}
```

## for in 과 for of 반복문

|**구분**|**for...in**|**for...of**|
|---|---|---|
|**주요 대상**|**객체(Object)**|**반복 가능한 객체(Array, Set, Map 등)**|
|**반복 내용**|열거 가능한 모든 **속성명(Key/Index)**|데이터 자체의 **값(Value)**|
|**비고**|상속받은 프로토타입 속성도 포함될 수 있음|실제 배열 데이터 순회에 가장|
```js
let arr = [10, 20];
arr.name = "custom";

// for...in : "0", "1", "name" (인덱스와 추가 속성까지 다 훑음)
// for...of : 10, 20 (배열의 실제 값만 깔끔하게 출력)
```

```js
let arr = [3,4,6]

 *   arr.foo = "hello";

 *

 *  for (let key in arr) { console.log(key)} // "0", "1", "2", "foo"  //속성의 key값을 통해 반복함

 *  for (let value of arr) {console.log(value)} // "3", "4", "6"  //속성의 값을 통해 반복함
```
### for in

 객체의 열거된 속성(key값)을 지정된 변수를통해서 모두 반복합니다.
 ```js
  for (let key in obj) {코드수행}
 ```
단, 객체 자신의 속성만 확인하려면 `Object.keys()`와 조합하는 것이 좋습니다.
### for of
iterable객체(반복가능한객체-array, map, set, arguments-등)을 반복하는 루프.

iterable객체의 값을 반복합니다.
```js
for (let value of arr) {코드수행}
```

### 💡 요약 및 권장 사항

1. **배열의 값을 순회할 때:** `for...of`를 쓰세요. 가장 안전하고 직관적입니다.
2. **배열의 인덱스가 필요할 때:** 기본 `for`문이나 `Array.prototype.forEach()`를 쓰세요.
3. **객체의 속성을 훑을 때:** `for...in`을 쓰세요. (단, 객체 자신의 속성만 확인하려면 `Object.keys()`와 조합하는 것이 좋습니다.)
    

## forEach반복문
`forEach`는 배열(Array) 객체에서 제공하는 **메서드(Method)**로, 배열의 모든 요소를 하나씩 순회하며 전달받은 **콜백 함수**를 실행합니다. 

앞서 보신 `for`문보다 훨씬 간결하고 가독성이 좋아 실무에서 가장 선호되는 방식 중 하나입니다.

### 기본 문법

`forEach`는 인자로 **함수**를 받으며, 그 함수는 최대 3개의 매개변수를 가질 수 있습니다.

```
배열.forEach(function(item, index, array) {
  // 실행할 코드
});
```

1. **item (value):** 현재 순회 중인 요소의 값
2. **index:** 현재 요소의 인덱스 (선택 사항)
3. **array:** `forEach`를 호출한 배열 자체 (선택 사항)

```js
const fruits = ['사과', '바나나', '포도'];

// 화살표 함수를 사용한 간결한 표현
fruits.forEach((fruit, i) => {
  console.log(`${i + 1}번째 과일: ${fruit}`);
});

/* 출력 결과:
1번째 과일: 사과
2번째 과일: 바나나
3번째 과일: 포도
*/
```

## ⚠️ forEach만의 특징 (중요!)

### ① break와 continue를 사용할 수 없습니다.

`for`문이나 `while`문과 달리, `forEach`는 루프 중간에 **멈출 수 없습니다.**

- 전체 요소를 무조건 끝까지 순회해야 합니다.
- 중간에 멈춰야 한다면 `for...of`나 `every()`, `some()` 같은 다른 배열 메서드를 써야 합니다.
    

### ② 오직 '배열(Array)'에서만 동작합니다.

`for...in`은 객체에도 쓸 수 있고, `for...of`는 문자열 등에도 쓸 수 있지만, `forEach`는 **배열 전용**입니다. (단, NodeList 같은 유사 배열 객체도 일부 지원합니다.)

### ③ 반환값(Return value)이 항상 `undefined`입니다.

`map`이나 `filter`와 달리, 실행 결과를 모아서 새로운 배열을 만들지 않습니다. 단순히 **"반복 작업을 수행"**하는 것이 목적입니다.
