# vim 설치
Ubuntu / Debian / Kali Linux:
```
sudo apt update
sudo apt install vim
```
설치가 완료되었다면 터미널(또는 CMD)에서 버전을 확인해 보세요.

```
vim --version
```

# vim 호환성모드 확인
**호환성 모드(Compatible Mode):** 리눅스 배포판에 따라 `vi` 명령어가 아주 기본적인 기능만 갖춘 구형 모드로 실행될 때가 있습니다. 이 경우 키 입력이 꼬일 수 있습니다.

신 버전의 Vim이라도 설정 파일(`.vimrc`)이 없으면 하위 호환성을 위해 `vi`처럼 동작하는 'Compatible Mode'로 작동할 수 있습니다. 이를 확인하려면 Vim 안에서 다음을 입력해 보세요.

1. `Esc` (또는 `Ctrl + [`)를 눌러 명령 모드로 이동합니다.
2. `:set compatible?` 를 입력하고 `Enter`를 누릅니다.
    - `compatible`이라고 뜨면: 구형 vi 호환 모드 (방향키, Esc 오류 발생 가능)
    - **`nocompatible`**이라고 뜨면: 현대적인 Vim 모드 (정상)


# vim 설정

최신 버전의 기능을 온전히 쓰려면 사용자 홈 디렉토리에 설정 파일을 만들어주는 것이 가장 깔끔합니다.



## wsl 전용 설정
```
# 1. 설정 파일 생성 및 nocompatible 모드 활성화 (현대적 Vim 모드)
echo "set nocompatible" > ~/.vimrc

# 2. 백스페이스가 정상적으로 작동하도록 설정
echo "set backspace=indent,eol,start" >> ~/.vimrc

# 3. 마우스 클릭으로 커서 이동 가능하게 설정 (선택사항이지만 WSL에서 매우 편리함)
echo "set mouse=a" >> ~/.vimrc
```

# Vim의 핵심: 3가지 모드 및 단축키

Vim은 메모장처럼 바로 글을 쓰는 게 아니라, 모드를 전환하며 사용합니다.

## 명령모드
- **명령 모드 (Normal Mode):** 시작 시 기본 모드로 명령모드로 시작합니다.  
	- 해당모드에서 복사, 삭제, 이동을 합니다. (**`Esc`** 키로 진입)

 ✍️ 편집하기기
- **`x`**: 커서가 있는 글자 한 개 삭제
- **`dd`**: 현재 줄 전체 삭제 (잘라내기)
- **`yy`**: 현재 줄 복사
- **`p`**: 붙여넣기
- **`u`**: 되돌리기 (Undo)

🔍이동하기기
- 방향키를 써도 되지만, 고수들은 `h`(왼), `j`(아래), `k`(위), `l`(오른)을 씁니다.
	- 위로 스크롤 `k`, 아래로 스크롤`j'
- **`gg`**: 문서 맨 처음으로 이동
- **`G`**: 문서 맨 끝으로 이동
- **`/단어`**: 특정 단어 검색 (`n`을 누르면 다음 검색어로 이동)

## 입력모드
    
- **입력 모드 (Insert Mode):** 실제 글자를 입력하는 모드. (명령 모드에서 **`i`** 입력)
	- insert모드로 진행시 좌측 하단에 `--insert--` 가 표기됩니다. 
	- esc키를 누르면 insert모드에서 빠져나와 명령모드로 진입합니다. 
	- 입력모드에서는 입력만 가능합니다. 

- - **`i`**: 현재 커서 위치에서 입력 시작

## 명령행모드
- **명령행 모드 (Command-line Mode):** 저장, 종료, 검색을 합니다. (명령 모드에서 **`:`** 입력)

- `:q` – 종료 (수정 사항이 없을 때)
    
- `:q!` – 저장하지 않고 강제 종료
    
- `:w` – 저장
    
- **`:wq` – 저장 후 종료** (가장 많이 씀)
## esc키가 먹히지 않는 경우
`Esc` 키를 눌러도 명령 모드로 나가지지 않고 화면에 `^[` 같은 이상한 문자만 출력되어 당황하셨군요!

이 현상은 주로 **Vim이 아닌 'Vi' (기본형) 모드**로 동작 중이거나, 터미널 환경 설정 문제로 인해 에디터가 현재 **입력 모드(Insert Mode)**에 갇혀 있을 때 발생합니다.

`Esc`가 먹히지 않는다면 다음 키 조합을 시도해 보세요.

- **`Ctrl + [`**: Vim에서 `Esc`와 동일한 역할을 하는 표준 단축키입니다. (가장 확실한 방법)
    
- **`Ctrl + c`**: 현재 진행 중인 입력을 취소하고 명령 모드로 강제 복귀합니다.

명령 모드로 돌아왔다면 하단의 `-- INSERT --` 글자가 사라집니다. 그 상태에서 `:wq`를 입력해 저장하고 나오시면 됩니다.

> 이미 명령모드에 진입한경우에 esc키를 눌러도 반응하지않습니다. 



# 커스터마이징 플러그인

## 플러그인 관리자: vim-plug 설치

Vim에서 플러그인을 가장 쉽고 빠르게 관리할 수 있는 도구는 **vim-plug**입니다. WSL 터미널에 아래 명령어를 복사하여 입력하세요.

```
curl -fLo ~/.vim/autoload/plug.vim --create-dirs \
    https://raw.githubusercontent.com/junegunn/vim-plug/master/plug.vim
```

## `.vimrc` 설정 및 플러그인 추가
설치 후에는 `~/.vimrc` 파일을 열어 사용할 플러그인을 목록에 등록해야 합니다.

```
" ~/.vimrc 파일 내용
call plug#begin('~/.vim/plugged')

" 여기에 플러그인을 추가하세요
Plug 'preservim/nerdtree'          " 파일 트리 탐색기
Plug 'tpope/vim-commentary'       " 간편한 주석 처리
Plug 'vim-airline/vim-airline'    " 하단 상태바 예쁘게 꾸미기
Plug 'morhetz/gruvbox'            " 눈이 편한 테마 (추천)
Plug 'junegunn/fzf.vim'           " 강력한 파일 검색

call plug#end()

" 테마 적용 (플러그인 설치 후)
autocmd vimenter * colorscheme gruvbox
set background=dark
```

## 입문자 필수 플러그인 추천
WSL 환경에서 개발 효율을 높여주는 핵심 플러그인들입니다.

| **플러그인 명**         | **기능 설명**                            | **[cite]** |
| ------------------ | ------------------------------------ | ---------- |
| **NERDTree**       | 좌측에 파일 디렉토리 구조를 보여주어 마우스 없이 파일 이동 가능 |            |
| **vim-airline**    | 현재 모드, 파일 경로, 인코딩 등을 세련되게 표시하는 상태바   |            |
| **gruvbox**        | 가독성이 뛰어나 고수들이 가장 많이 사용하는 다크 테마 중 하나  |            |
| **fzf.vim**        | 파일명이나 파일 내용을 아주 빠르게 검색해주는 도구         |            |
| **Vim-commentary** | `gcc` 입력 시 한 줄 주석, `gc`로 블록 주석 처리 가능 |            |
**아이콘 표시:** 만약 Airline 같은 플러그인에서 아이콘이 깨져 보인다면, 윈도우에 [Nerd Fonts](https://www.nerdfonts.com/)를 설치하고 윈도우 터미널 글꼴을 해당 폰트로 바꿔야 합니다.
## 플러그인 설치 실행 방법

1. `.vimrc` 파일을 저장하고 나옵니다.
    
2. 다시 `vim`을 실행합니다.
    
3. 명령 모드에서 **`:PlugInstall`**을 입력하고 `Enter`를 누릅니다.
    
4. 왼쪽에 설치 진행 상황이 뜨며, 완료되면 `q`를 눌러 닫습니다.
