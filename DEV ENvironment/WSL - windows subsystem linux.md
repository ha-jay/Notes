# wsl 접속법 - 여러 version의 wsl 존재시

```powershell
## default wsl로 접속
wsl
## 특정 distribution으로 접속(1회성)
wsl -d ubuntu
## 특정 wsl을 default로 변경하기
wsl --set-default ubuntu
```

`wsl -l -v` 명령어를 쳤을 때 `ubuntu` 외에 `docker-desktop`이 목록에 나타나는 것은 지극히 정상적인 현상입니다.
![[Pasted image 20260203150635.png]]
이는 Docker Desktop이 윈도우에서 리눅스 컨테이너를 실행하기 위해 자기만의 전용 '미니 리눅스(Distro)'를 WSL 안에 몰래 설치해 두었기 때문입니다. 
현재 `docker-desktop`이 기본값(Default)으로 설정되어 있어서 그냥 `wsl`만 치면 그쪽으로 들어가는 것이죠.

상황에 따라 **Ubuntu**로 접속하는 세 가지 방법을 깔끔하게 정리해 드릴게요.

### 1. 명령어 한 번으로 Ubuntu 접속하기 (일회성)

기본 설정을 바꾸지 않고 바로 Ubuntu로 들어가고 싶을 때는 `-d` (distribution) 옵션을 사용합니다.

- **명령어:** `wsl -d ubuntu`
- **핵심 이론:** WSL은 여러 개의 리눅스를 동시에 가질 수 있는 아파트와 같습니다. `d` 옵션은 "그중에서 특정 호수(배포판)로 바로 들어가겠다"는 지정석 티켓과 같습니다.

### 2. Ubuntu를 '기본 리눅스'로 고정하기 (추천)

매번 `-d`를 치기 귀찮다면, Ubuntu를 기본값으로 설정하면 됩니다. 앞으로 `wsl`만 치면 바로 Ubuntu가 열립니다.

- **명령어:** `wsl -s ubuntu` (또는 `wsl --set-default ubuntu`)
- **확인 방법:** `wsl -l -v`를 쳤을 때 `ubuntu` 이름 옆에 표시가 붙어 있다면 성공입니다.
![[Pasted image 20260203150338.png]]
### 3. VS Code에서 Ubuntu 골라서 열기 (실무용)

우리는 주로 VS Code에서 작업을 할 테니, 이 방법이 가장 중요합니다.

1. 윈도우에서 **VS Code**를 실행합니다. / WSL EXTENSION 설치
2. 키보드에서 **`F1`** 키를 누릅니다 (명령 팔레트).
3. *`WSL: Connect to WSL using Distro...`*를 검색해서 선택합니다.
4. 목록에 뜨는 배포판 중 **`Ubuntu`*를 클릭합니다.
5. 그러면 VS Code 왼쪽 하단에 `WSL: Ubuntu`라고 뜨면서 접속됩니다.

### [운영 팁] 왜 Docker 배포판이 따로 있나요?

Docker Desktop은 `docker-desktop`과 `docker-desktop-data`라는 두 개의 배포판을 관리합니다.

- **`docker-desktop`**: 도커 엔진 자체를 돌리는 시스템 영역입니다.
- **`docker-desktop-data`**: 우리가 나중에 만들 컨테이너 이미지나 데이터가 저장되는 창고 영역입니다.

우리는 이 두 곳에 직접 들어가서 코딩하는 것이 아니라, **우리의 작업실인 `ubuntu`** 안에서 도커 명령어를 내려서 저 엔진들을 부려먹는 구조로 학습하게 될 것입니다.

이제 `wsl -s ubuntu` 명령어로 기본값을 바꾸신 다음, `wsl` 명령어로 Ubuntu에 접속해 보세요. 접속 후 **`whoami`**를 입력했을 때 아까 설정한 리눅스 사용자 이름이 나온다면 완벽히 준비된 것입니다!

# 윈도우와 WSL 파일 시스템

우리가 구축할 프로그램의 성능과 안정성은 이 '파일 저장소'를 어떻게 활용하느냐에 달려 있다고 해도 과언이 아닙니다.

WSL 2 시스템의 파일 저장소 구조를 '두 개의 세계(윈도우와 리눅스)'와 '연결 통로'라는 개념으로 쉽게 설명해 드릴게요.

### 1. 리눅스 전용 저장소 (가상 하드 디스크)

WSL 2는 윈도우 안에 **`.vhdx`**라는 확장자를 가진 가상 하드 디스크 파일을 하나 만듭니다. 이 파일이 바로 리눅스의 '내 집'이자 전용 저장소입니다.

- **파일 시스템:** 이 가상 디스크 안은 리눅스 전용인 **Ext4**라는 파일 시스템으로 포맷되어 있습니다.
- **경로:** 리눅스 터미널에서 `cd ~`를 입력하면 들어가는 `/home/유저명` 폴더가 바로 이 영역에 있습니다.
- **특징:** 리눅스 운영체제와 한 몸이기 때문에 읽고 쓰는 속도가 매우 빠릅니다.

### 2. 윈도우와의 연결 통로 (마운트)

WSL은 윈도우의 파일들도 리눅스에서 볼 수 있도록 '길'을 연결해 줍니다. 이것을 **마운트(Mount)**라고 부릅니다.

- **윈도우 -> 리눅스:** 리눅스 터미널에서 `/mnt/c/` 경로로 들어가면 윈도우의 C 드라이브 파일들을 그대로 볼 수 있습니다.
	![[Pasted image 20260203151609.png]]
- **리눅스 -> 윈도우:** 윈도우 파일 탐색기 주소창에 `\\wsl.localhost\\Ubuntu` 를 입력하면 리눅스 안의 파일들을 윈도우 화면에서 폴더처럼 볼 수 있습니다.
	![[Pasted image 20260203151717.png]]
### 3. [중요] 코드를 윈도우에 저장하고 wsl에서 실행하면?

여기서 초보자가 가장 많이 하는 실수가 나옵니다. 윈도우 폴더(`C:\\...`)에 코드를 저장하고 WSL로 실행하는 것입니다.

- **성능 저하:** 윈도우 영역의 파일을 리눅스가 읽으려면 네트워크 통신과 비슷한 복잡한 과정을 거쳐야 합니다. 이 경우 파일 읽기 속도가 **수십 배 이상 느려질 수 있습니다**.
- **권한 문제:** 리눅스 특유의 파일 권한(실행 권한 등)이 윈도우 영역에서는 제대로 작동하지 않아 에러가 날 수 있습니다.

### 지금 기억할 실천 지침 - 코드는 리눅스에 보관

1. **데이터베이스와 코드는 리눅스 영역에:** 앞으로 우리가 만들 메일 서버의 데이터나 게시판 소스 코드는 반드시 `/home/유저명/` 하위 폴더에 저장하세요.
	아래에 보이는 `~` 경로가 리눅스의 `home/유저명` 폴더입니다.
	![[Pasted image 20260203151959.png]]
2. **명령어 꿀팁:** 리눅스 터미널에서 현재 폴더를 윈도우 탐색기로 보고 싶다면 `explorer.exe.` 라고 입력해 보세요. 마법처럼 윈도우 폴더 창이 뜰 겁니다.

**요약하자면:** "WSL은 윈도우 안에 **가상 하드 디스크**를 만들어 리눅스 전용 공간을 확보하며, `/mnt/c/`라는 통로를 통해 윈도우와 소통한다. 하지만 속도를 위해 **모든 개발 작업은 리눅스 전용 공간(Home 디렉토리)에서 해야 한다**"는 것입니다.
