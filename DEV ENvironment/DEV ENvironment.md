

# 운영체제

## 리눅스

### WSL

- WSL 이란 윈도우 system 안에서 리눅스를 하드웨어 수준에서 효율적으로 쓸수있게 만들어주는 기술입니다. 윈도우는 우리가 평소에 쓰는 편리한 집이지만, 개발과 서버 운영의 세계에서는 **리눅스(Linux)** 라는 환경이 표준입니다. 리눅스를 설치하지않고도, 윈도우 안에 **'리눅스 전용 방'** 을 하나 만드는 기술입니다. 이 방 안에서는 진짜 리눅스 명령어를 쓰고 리눅스용 프로그램을 돌릴 수 있습니다.
- 보통 리눅스를 쓰려면 컴퓨터 전체를 리눅스로 바꾸거나, '가상 머신(VM)'이라는 무거운 프로그램을 돌려야 했습니다. 하지만 WSL 2는 윈도우 안에 **실제 리눅스 커널(심장)**을 직접 심어버린 방식입니다.
	- **속도가 빠릅니다:** 가상 머신처럼 컴퓨터 자원을 많이 잡아먹지 않고, 윈도우와 자원을 효율적으로 나눠 씁니다 .
	- **윈도우와 한 몸처럼 움직입니다:** 윈도우에서 작업하던 파일을 리눅스에서 바로 열어볼 수 있고, 그 반대도 가능합니다.
- 우리가 만들 프로젝트는 나중에 실제 운영 서버에 올릴 때 **리눅스 환경**에서 돌아갈 확률이 매우 높습니다 .
	- **배포 환경과 일치:** 윈도우에서 개발하고 리눅스 서버에 올리면 "내 컴퓨터에선 됐는데 서버에선 안 돼요!" 하는 문제가 자주 생깁니다. 하지만 WSL을 쓰면 내 노트북 자체가 이미 리눅스 서버와 똑같은 환경이 되므로 이런 문제를 미리 방지할 수 있습니다 .
	- **강력한 도구들:** 서버 운영 전문가들이 사용하는 도구(Docker, Nginx, 데이터베이스 등)는 리눅스에서 가장 완벽하게 돌아갑니다.

# Terminal & SCRIPTING & CLI Editor

터미널이란 GLI 인터페이스를 이용하여 다음의 다양한 역활을 수행합니다. 
- **운영체제와의 소통**: `ls` (파일 목록 보기), `cd` (디렉터리 이동)와 같은 기본적인 운영체제 명령어를 실행하여 파일 시스템을 관리합니다.
- **개발 도구 실행**: 프로그래밍 언어 컴파일러, 빌드 도구, 버전 관리 시스템(Git), docker등 다양한 개발 도구를 실행하는 데 사용됩니다.
- **서버 관리**: 원격 서버에 접속하여(SSH) 서버를 관리하고 제어할 때 필수적인 도구입니다.
실무에서 서버를 배포하거나, 클라우드(AWS 등)를 다루고, Docker 같은 도구를 쓸 때 모두 리눅스 명령어 기반으로 소통합니다.

처음에는 어려워 보일 수 있지만, 강력한 스크립트를 작성하여 여러 작업을 한 번에 처리하는 등 자동화에 매우 효율적입니다.



그래서 우분투 터미널(Bash/Zsh)을 익히는 것은 개발자의 **'공통 언어'**를 배우는 것과 같습니다.
