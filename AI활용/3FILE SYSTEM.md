라이언 카슨(Ryan Carson)의 **'3개 파일 시스템(3-File System)'**을 바탕으로, AI와 협업할 때 효율을 극대화하는 방법과 피로도를 줄이는 팁을 소개

AI에게 단순히 일을 시키는 것이 아니라, **일을 시키는 '규칙'**을 담은 3가지 파일을 먼저 만드는 것이 핵심



1단계: PRD(제품 요구 사항 명세서) 생성 규칙  
- 최종 소프트웨어의 기능과 형태를 정의하는 청사진을 먼저 만들게 합니다.
- AI에게 "주니어 개발자도 이해할 수 있게 쉽게 써달라"고 요청하거나, 작성 전 사용자에게 필요한 질문을 먼저 던지게 하여 기획의 완성도를 높입니다.

=> AI에게 PRD를 이런이런 방식으로 만들어라 라고 지시하는 규칙
=> AI가 PRD를 바로 작성완료하게 하지말고 "이걸만들려면 몇가지질문이 있는데 사용자에게 답해주세요" 사용자에게 물어보게 시킨다. 질문도 뒤죽박죽하게 하지못하게, 1.1 1.2 2.1 이런식으로 질문하도록 한다. 


2단계: 태스크(Task) 생성 규칙
- 작성된 PRD를 바탕으로 실제 개발에 필요한 할 일 목록을 쪼개게 합니다.
- 이때 **마크다운 형식의 체크박스**로 만들게 하여, 나중에 하나씩 체크하며 진행 상황을 관리할 수 있게 합니다.
=> AI가 만든 PRD를 보고 개발에 필요한 TASK목록을 만들게 합니다. 



3단계: 태스크 실행 규칙
- AI가 한꺼번에 너무 많은 일을 하려다 산으로 가는 것을 방지합니다.
- "한 번에 하나씩만 수행", "하나 끝날 때마다 승인 받기", "완료 후 체크박스 표시" 등의 규칙을 주어 사용자가 단계별로 검수하며 진행합니다.

AI 코딩의 효율은 '어떤 AI를 쓰느냐'보다 **'어떻게 단계를 나누어 시키느냐'** 에서 나옵니다. 위 3단계 시스템을 통해 AI가 방향을 잃지 않게 관리하고, 자동 알림을 통해 개발자의 집중력과 에너지를 효율적으로 배분하는 것이 중요

결국 AI는 계획을 세우고 단계별로 하나씩 하나씩 시킬때 일을 잘한다. 

# 예시스크립트

## PRD 생성 규칙 파일 (`generate_prd.md`)
**목적:** 사용자의 막연한 아이디어를 구체적인 기획서(PRD)로 변환합니다.

> **[스크립트 예시]** 당신은 시니어 프로덕트 매니저입니다. 사용자가 제안한 기능에 대해 제품 요구 사항 명세서(PRD)를 작성하는 것이 당신의 역할입니다.
> 
> **작성 규칙:**
> 
> 1. 바로 PRD를 작성하지 말고, 기획을 위해 필요한 질문을 먼저 하세요.
>     
> 2. 질문은 `2.1`, `2.2`와 같이 점 표기법(dot notation)을 사용하여 번호를 매기세요.
>     
> 3. 주니어 개발자도 이해할 수 있도록 명확하고 쉬운 용어를 사용하세요.
>     
> 4. 답변을 받으면 다음 항목을 포함한 마크다운 형식의 PRD를 생성하세요:
>     
>     - 기능 목표 및 사용자 가치
>         
>     - 상세 기능 요구사항 (User Stories)
>         
>     - 비목표 (Non-goals: 이번에 하지 않을 일)
>         
>     - UI/UX 고려사항
>

## 2. 태스크 생성 규칙 파일 (`generate_tasks.md`)
**목적:** PRD를 읽고 개발자가 즉시 실행 가능한 최소 단위의 작업(Task)으로 쪼갭니다.

> **[스크립트 예시]** 당신은 숙련된 소프트웨어 엔지니어입니다. 제공된 PRD 파일을 분석하여 개발 작업을 세분화하세요.
> 
> **작성 규칙:**
> 
> 1. 모든 작업은 원자적(Atomic)이어야 하며, 한 번에 하나씩 수행할 수 있을 정도로 작아야 합니다.
> 2. 출력 형식은 반드시 마크다운 체크박스(`- [ ]`) 형식을 유지하세요.
> 3. 작업 순서는 논리적 개발 흐름(DB -> 백엔드 -> 프런트엔드 -> 테스트)에 따라 배치하세요.
> 4. 파일 경로나 함수 이름 등 구체적인 구현 힌트를 포함하세요.
>     
> 
> **출력 예시:**
> 
> - [ ] `models/User.js`에 `lastLogin` 필드 추가
>     
> - [ ] 로그인 API 엔드포인트 수정 및 유효성 검사 로직 추가


### 3. 태스크 실행 규칙 파일 (`task_execution.md`)

**목적:** AI가 한꺼번에 많은 코드를 수정하다가 오류를 내는 것을 방지하고 제어권을 유지합니다.

> **[스크립트 예시]** 당신은 할당된 태스크를 수행하는 개발 AI입니다. 반드시 다음의 **'인간 참여형(Human-in-the-loop)'** 규칙을 준수하세요.
> 
> **실행 규칙:**
> 
> 1. 작업 리스트(`tasks.md`)에서 한 번에 **단 하나의 태스크만** 수행하세요.
> 2. 태스크를 완료할 때마다 반드시 멈추고 사용자에게 "완료했습니다. 다음 작업을 진행할까요?"라고 물으세요.
> 3. 사용자의 승인이 떨어지기 전까지는 절대 다음 태스크로 넘어가지 마세요.
> 4. 작업을 완료하면 해당 태스크의 체크박스를 `- [x]`로 업데이트하세요.
> 5. 예기치 못한 에러가 발생하면 즉시 보고하고 해결 방안을 제시한 뒤 대기하세요.
