실제 프로젝트를 하나 상정하고 시나리오를 그려볼게요. 예시로 **"SaaS 웹 애플리케이션 (할일 관리 + 팀 협업 툴)"** 을 만든다고 가정하겠습니다.

## 🗺️ 프로젝트 AI 활용 전체 흐름

---

### 1단계 — 기획 & 아키텍처 설계

**사용 도구: Claude / ChatGPT**

아직 코드를 한 줄도 쓰기 전, 머릿속에 있는 아이디어를 구체화하는 단계입니다.

```
나: "팀 협업 할일 관리 SaaS를 만들려고 해. 
     유저 인증, 프로젝트별 칸반보드, 실시간 알림이 필요해.
     기술 스택 추천과 전체 아키텍처를 그려줘."
```

Claude가 해주는 것들:

- 기술 스택 추천 (Next.js + Supabase + Redis 등) 및 각 선택의 트레이드오프 설명
- ERD 초안 (테이블 구조, 관계)
- API 엔드포인트 목록 초안
- 예상 병목 지점 및 주의사항

**포인트:** 이 단계에서 AI와 여러 번 대화하며 설계를 다듬습니다. 혼자 며칠 걸릴 설계 검토를 몇 시간으로 단축할 수 있습니다.

---

### 2단계 — 프로젝트 세팅 & 보일러플레이트

**사용 도구: Claude Code**

설계가 확정되면 초기 세팅은 반복 작업의 연속입니다. 에이전트에게 맡기는 게 효율적입니다.

bash

````bash
# 터미널에서
claude "Next.js 14 + TypeScript + Supabase + Tailwind CSS로 
        프로젝트 초기 세팅해줘. ESLint, Prettier, Husky도 설정하고
        폴더 구조는 feature-based로 잡아줘"
```

Claude Code가 해주는 것들:
- 패키지 설치 및 설정 파일 생성
- 폴더 구조 자동 생성
- 환경변수 파일 템플릿 작성
- Git 초기화 및 .gitignore 설정

**포인트:** 개발자가 직접 하면 1~2시간 걸리는 세팅을 10분 안에 끝냅니다.

### 3단계 — 핵심 기능 개발
**사용 도구: Cursor (메인) + Claude (서브)**

본격적인 개발 단계입니다. 여기서 가장 많은 시간을 씁니다.

**Cursor 활용 패턴:**
```
// 유저 인증 구현
Cursor에게: "Supabase Auth로 이메일/소셜 로그인 구현해줘.
             미들웨어로 라우트 보호도 추가하고,
             세션 관리는 /lib/auth.ts에 분리해줘"
```

Cursor가 코드베이스 전체를 이해하고:
- 관련 파일들을 한 번에 수정
- 기존 코드 스타일에 맞춰 생성
- 타입 정의도 자동으로 추가

**Claude 서브 활용:**
복잡한 비즈니스 로직이나 알고리즘을 짤 때는 채팅으로 먼저 설계를 잡습니다.
```
나: "칸반 보드에서 드래그앤드롭으로 카드 순서를 바꿀 때
     DB에 순서를 어떻게 저장하는 게 효율적일까? 
     (Lexorank vs 정수 인덱스 비교해줘)"
```

설계 결정을 내린 후 Cursor에서 구현합니다.
````

## 💡 핵심 원칙 3가지

**1. AI는 운전대가 아니라 네비게이션입니다.** 방향을 결정하는 건 개발자입니다. AI가 만든 코드를 이해 없이 붙여넣으면 나중에 반드시 큰 문제가 생깁니다.

**2. 컨텍스트를 많이 줄수록 결과가 좋아집니다.** "버그 고쳐줘"보다 "이런 환경에서, 이런 목적으로 만든 코드인데, 이런 에러가 나"처럼 상황을 풍부하게 주면 답의 품질이 크게 달라집니다.

**3. 단계를 잘게 쪼개야 합니다.** AI에게 "앱 전체 만들어줘"는 잘 안 됩니다. "인증 모듈만", "이 컴포넌트만"처럼 작은 단위로 요청할수록 결과물이 정확합니다.
