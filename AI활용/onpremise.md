## Phase 2 — Linux + 네트워크 (온프레미스의 뿌리)

온프레미스는 결국 **"내가 직접 서버를 관리한다"** 는 것입니다. Linux와 네트워크를 모르면 아무것도 할 수 없어요.

### Linux 서버 구성

```
배우는 순서:
1. 기본 명령어 (파일, 권한, 프로세스 관리)
2. 서비스 관리 (systemd, 부팅 시 자동 실행)
3. 사용자/그룹 관리 (보안 환경에서 매우 중요)
4. 로그 관리 (/var/log, journalctl)
5. 스크립트 자동화 (Bash scripting)
```

**실습 환경 꿀팁 — 돈 안 들이고 만들기:**

```
방법 1: VirtualBox (무료) + Ubuntu Server 설치
         → 내 PC에 가상 서버 여러 대 만들기

방법 2: Multipass (무료)
         → 명령어 하나로 Ubuntu VM 즉시 생성

방법 3: Raspberry Pi
         → 실제 물리 서버 경험 (3~5만원)
```

**AI 활용법:**

```
"Ubuntu Server를 처음 설치했어. 
 보안 강화를 위해 초기에 해야 할 설정들을 
 순서대로 알려줘 (SSH 키 인증, 방화벽, 불필요한 서비스 제거 등)"

"systemd로 내 Node.js 앱을 서비스로 등록해서 
 서버 재시작 시 자동으로 켜지게 하고 싶어"
```

### 네트워크 (보안 환경의 핵심)

보안 제약 환경일수록 네트워크 설계가 중요합니다.

```
배우는 순서:
1. 기본 개념
   IP, 서브넷, CIDR, Gateway, DNS 동작 원리

2. 방화벽
   iptables / nftables (Linux 내장)
   UFW (Ubuntu 간편 방화벽)
   → 포트 제어, 화이트리스트 정책

3. DNS 서버 직접 구축
   BIND9 or CoreDNS
   → 내부망에서 도메인 직접 관리

4. VPN
   WireGuard (현대적, 빠름) — 추천
   OpenVPN (전통적, 기업 많이 씀)
   → 원격 접속, 사이트 간 연결

5. 네트워크 분리 (VLAN)
   → 개발망 / 운영망 / 관리망 분리
   → 보안 환경에서 필수 개념
```

**AI 활용법:**

```
"금융권처럼 인터넷 차단된 내부망 환경에서 
 개발서버 / 운영서버 / DB서버를 
 네트워크적으로 어떻게 분리해야 해?
 VLAN 설계 예시 그려줘"

"WireGuard로 사무실 내부망에 
 외부에서 안전하게 접속하는 VPN 구성해줘"
```

## Phase 3 — 가상화 (서버 자원 효율화)

물리 서버 1대로 여러 서버처럼 운영하는 기술입니다. 온프레미스 환경에서 핵심이에요.

### Proxmox (추천 — 무료 + 강력)

```
배우는 순서:
1. Proxmox 설치 및 웹 UI 이해
2. VM (가상머신) 생성 및 관리
3. LXC 컨테이너 (VM보다 가볍고 빠름)
4. 스냅샷 / 백업 자동화
5. 클러스터 구성 (서버 여러 대 묶기)
6. HA (고가용성) — 서버 1대 죽어도 서비스 유지
```

**실습 환경:** 중고 미니PC (인텔 NUC 등, 10~20만원대) 1~2대로 실제 Proxmox 클러스터 구성 가능합니다. 이게 가장 실전에 가까운 학습이에요.

### VMware (기업 표준)

```
vSphere / ESXi — 대기업, 공공기관 표준
vCenter — 여러 ESXi 서버 중앙 관리
```

무료 실습은 VMware Workstation Pro (개인 무료 전환됨)로 가능합니다.

**AI 활용법:**

```
"Proxmox에서 Ubuntu Server VM 3대로 
 웹서버 1대 + DB서버 1대 + 백업서버 1대 구성을 
 처음부터 설명해줘"

"Proxmox HA 클러스터를 구성할 때 
 최소 몇 대의 서버가 필요하고 왜 그런지 설명해줘"
```

## Phase 5 — OpenStack (프라이빗 클라우드)

AWS를 내 서버실에 직접 구축하는 것입니다. 규모 있는 기업/공공기관에서 많이 씁니다.

```
배우는 순서:
1. OpenStack 아키텍처 이해
   (Nova, Neutron, Cinder, Keystone... 컴포넌트가 많음)

2. DevStack으로 개발용 단일 노드 설치
   → 실제 배포 전 학습용

3. Kolla-Ansible로 실제 클러스터 배포
   → 컨테이너 기반 설치 (현재 표준)

4. 네트워크 구성 (Neutron)
   → 테넌트 네트워크, 플로팅 IP

5. 스토리지 (Ceph 연동)
   → 분산 스토리지, 온프레미스 S3 역할
```

**주의:** OpenStack은 진입장벽이 높습니다. Proxmox + Docker + Kubernetes를 먼저 익힌 후 도전하는 게 현실적이에요.

## 보안 제약 환경 특화 — 꼭 알아야 할 것들

일반 DevOps와 다르게 보안 환경에서 추가로 필요한 개념들입니다.

```
에어갭 환경 대비
├── 인터넷 없이 패키지 설치 (로컬 미러 서버 구축)
├── 프라이빗 Docker Registry (Harbor)
├── 내부 Git 서버 (Gitea, GitLab Self-hosted)
└── 오프라인 CI/CD 파이프라인

보안 강화
├── SELinux / AppArmor (강제 접근 제어)
├── 감사 로그 (auditd)
├── 취약점 스캐너 (OpenVAS)
└── 접근 제어 (PAM, sudo 정책)

컴플라이언스
├── CC인증 / ISMS 기본 개념
└── 망분리 아키텍처 설계
```

**AI 활용법:**

```
"인터넷이 완전 차단된 환경에서 
 Ubuntu 패키지 설치를 위한 
 내부 apt 미러 서버를 구축하는 방법 알려줘"

"Harbor로 프라이빗 Docker Registry 구축하고 
 내부 Kubernetes 클러스터와 연결하는 방법 알려줘"
```

## 클라우드 vs 온프레미스 비교하며 배우기

같은 걸 두 환경에서 구현해보는 게 가장 효과적입니다.

```
AWS EC2          ↔   Proxmox VM
AWS RDS          ↔   직접 설치한 PostgreSQL
AWS S3           ↔   MinIO (오픈소스 S3 호환)
AWS VPC          ↔   VLAN + 방화벽
AWS Route53      ↔   BIND9 DNS 서버
EKS              ↔   Self-hosted Kubernetes (k3s)
AWS ECR          ↔   Harbor Registry
```

같은 개념이 클라우드에서는 버튼 하나로 되는 걸, 온프레미스에서는 직접 구성해보면 **인프라의 본질**이 보입니다.

보안 제약 환경을 다루는 곳들입니다.

```
금융권 (은행, 증권, 보험)   망분리 필수, 온프레미스 비중 높음

의료기관                     개인정보 보호로 클라우드 제한

```

이 분야에서 **Linux + 네트워크 + 가상화 + 보안**을 모두 다룰 수 있는 사람은 정말 드물어서 희소성이 높습니다.
