# 표현식
표현식이란 어떤 하나의 값으로 평가될수 있는 것을 말한다.

변수에는 값을 할당할수 있다.
값으로 평가될수 있는 표현식은 변수에 담을수있다.
값으로 평가될수 없는 표현식은 변수에 담을수 없다.

선언문은 표현식이아니고 statements이다.

```js
let x = 1;
function f (){return "hello"}

console.log(1); // 값
console.log(1+1); //값의 연산
console.log(x); //변수 x의 값은 1이다.
console.log(x+3) //변수와 연산
console.log(x = 4); //할당연산자는 할당한 값을 반환한다. 4
console.log(x=x+3); //7반환

console.log(f()); //hellow
console.log(f()+" myname") //hello myname
console.log(f +"11");  //function f (){return "hello"}11
console.log(String+"55") //function String() { [native code] }55
```
표현식(Expression)은 하나의 값으로 평가(Evaluation)된다.

값(리터럴), 변수, 객체의 프로퍼티, 배열의 요소, 함수 호출, 메소드 호출, 피연산자와 연산자의 조합은 모두 표현식이며 하나의 값으로 평가(Evaluation)된다.

표현식은 결국 하나의 값이 되기 때문에

다른 표현식의 일부가 되어 조금 더 복잡한 표현식을 구성할 수도 있다.

### 단순한 표현식들

각각은 독립적으로 하나의 값을 가집니다.
- `10` (숫자 리터럴 → 값: **10**)
- `'Hello'` (문자 리터럴 → 값: **'Hello'**)
- `age` (변수 참조 → 값: **변수에 담긴 값**)
- `Math.random()` (함수 호출 → 값: **생성된 난수**)

### 표현식의 조합 (Building Blocks)

작은 표현식들이 모여 더 큰 표현식을 구성하는 예시입니다.
```js
// 예시: 5 + (10 * 2)
// 1. 10 은 표현식이다.
// 2. 2 는 표현식이다.
// 3. 10 * 2 는 두 표현식이 연산자로 결합된 더 큰 표현식이다 (값: 20)
// 4. 5 + 20 은 최종적으로 하나의 값 '25'가 되는 표현식이다.

let result = 5 + (10 * 2);
```
더 복잡한 표현식
```js
let user = { 
  baseAge: 20, 
  getAdd: (a, b) => a + b 
};

// 아래 전체 문장은 결국 하나의 숫자 '25'로 평가되는 표현식입니다.
let finalValue = user.getAdd(user.baseAge, 5);
```

- **표현식**은 마치 **계산기**에 입력했을 때 최종 결과값이 나오는 모든 코드 조각입니다.
    
- 자바스크립트 어디에서든 **"값이 들어갈 자리"**라면 그 자리에 **표현식**을 넣을 수 있습니다

# 연산자
## 이항연산자
앞과 뒤 2개의 피연산자를 필요로 하는 연산자

- **할당 연산자 (`=`):** 우항의 값을 좌항의 변수에 할당합니다.
- **산술 연산자:** `+`, `-`, `*`, `/`, `%`(나머지), `**`(거듭제곱)
- **복합 할당 연산자:** `+=`, `-=`, `*=`, `/=`, `%=`, `**=` 등 (예: `x += y` → `x = x + y`)
    
- **비교 연산자 (Boolean 반환):** * `==`, `!=`: 값만 비교 (타입 자동 변환 발생)
    - `===`, `!==`: **값과 타입**까지 엄격하게 비교 **(권장)**
    - `>`, `<`, `>=`, `<=`: 크기 비교
        
- **논리 연산자:**
    - `&&` (AND): 모두 참일 때 참
    - `||` (OR): 하나만 참이어도 참
    - `&&=`, `||=`: 논리 복합 할당 (ES2021 도입)

## 단항연산자
- **증감 연산자:** 
	* `++a` (전위): 증가 후 할당
    - `a++` (후위): 할당 후 증가
        
- **논리 부정 (`!`):** 참을 거짓으로, 거짓을 참으로 반환
- **기타:**
    - `delete`: 객체의 속성 삭제
    - `typeof`: 데이터 타입 확인
    - `void`: 표현식을 평가하되 값을 반환하지 않음

## 관계연산자
`in`: 객체 내 특정 키(property) 존재 여부 확인
`instanceof`: 특정 생성자 함수의 인스턴스인지 확인

## 삼항연산자
 `조건 ? 참일_때_값 : 거짓일_때_값` (간단한 조건문에 활용)

## 논리복합할당 연산자
논리 할당 연산자는 논리 연산(`&&`, `||`, `??`)과 할당 연산(`=`)을 결합한 형태입니다. 

핵심은 **"좌항의 조건이 충족될 때만 우항을 할당한다"**는 점입니다.

> **참고:** Node.js 15버전 이상, 최신 브라우저 환경에서 지원됩니다.
### 1. AND 할당 연산자 (`&&=`)

- **동작 원리:** 좌항(`x`)이 **참(Truthy)**일 때만 우항(`y`)을 할당합니다.
-  (x &&= y)
- **풀이:** `x && (x = y)`
    
- **용도:** 변수에 값이 이미 있을 때만 새로운 값으로 업데이트하고 싶을 때 사용합니다.
```js
let a = 1;
a &&= 2; // a가 1(참)이므로 2를 할당
console.log(a); // 2

let b = 0;
b &&= 2; // b가 0(거짓)이므로 아무 일도 일어나지 않음
console.log(b); // 0
```


### 2. OR 할당 연산자 (`||=`)

- **동작 원리:** 좌항(`x`)이 **거짓(Falsy)**일 때만 우항(`y`)을 할당합니다.
- (x ||= y)
- **풀이:** `x || (x = y)`
    
- **용도:** 변수가 비어있거나(`""`, `0`, `false` 등) 없을 때 기본값을 넣고 싶을 때 사용합니다.
```js
const config = { duration: 50, title: '' };

config.duration ||= 10; // 50은 참이므로 유지
console.log(config.duration); // 50

config.title ||= 'Untitled'; // ''은 거짓이므로 'Untitled' 할당
console.log(config.title); // "Untitled"
```

### Nullish 할당 연산자 (`??=`)

- **동작 원리:** 좌항(`x`)이 **null** 또는 **undefined**일 때만 우항(`y`)을 할당합니다.
- (x ??= y)    
- **풀이:** `x ?? (x = y)`
    
- **용도:** `0`이나 `""`(빈 문자열)은 유효한 데이터로 취급하고 싶고, 오직 값이 **아예 없을 때만** 기본값을 주고 싶을 때 매우 유용합니다.

```js
const data = { count: 0, text: "" };

data.count ??= 10; // count가 0이지만 null/undefined가 아니므로 유지
console.log(data.count); // 0 (||= 였다면 10이 되었을 것)

data.speed ??= 25; // speed는 undefined이므로 25 할당
console.log(data.speed); // 25
```

### 💡 실무 한 줄 팁

현대적인 자바스크립트 작성 시, 단순히 데이터 유무를 체크해서 기본값을 넣을 때는 `||=`보다 **`??=`**을 더 권장합니다. 사용자 입력값 `0`이나 빈 문자열 `""`이 의도치 않게 기본값으로 덮어씌워지는 버그를 방지할 수 있기 때문입니다.
