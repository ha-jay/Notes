## Truthy & Falsy (참 같은 값 & 거짓 같은 값)

자바스크립트 조건문에서는 불리언(`true/false`)이 아니더라도 참/거짓으로 평가되는 값들이 있습니다.

- **Falsy (거짓으로 평가):** `false`, `0`, `""`(빈 문자열), `null`, `undefined`, `NaN`
- **Truthy (참으로 평가):** 위 항목들을 제외한 모든 것 (빈 배열 `[]`, 빈 객체 `{}` 포함)


## 💡조건문  어떤 것을 써야 할까요?

1. **간단한 true/false 구분:** `if ... else`
2. **한 줄로 결과값 할당:** 삼항 연산자
3. **일치하는 값이 많은 경우:** `switch`
4. **객체의 존재 여부 확인 후 실행:** `&&` 단락 평가

# 조건문
##  if ... else 문

가장 기본적이고 많이 쓰이는 조건문입니다.

if문은 2가지선택지가있고(선택지가 많지않고),
조건이 복잡하고(and or연산자..등), 
각각 조건에따라 수행되어야할 코드가 많은경우에 사용한다.

- **if**: 조건이 참일 때 실행
- **else if**: 앞의 조건이 거짓이고, 새로운 조건이 참일 때 실행
- **else**: 모든 조건이 거짓일 때 실행
```js
let distance = 15;

if (distance < 10) {
  console.log("걸어가세요.");
} else if (distance < 30) {
  console.log("버스를 타세요.");
} else {
  console.log("택시를 타세요.");
}
```

### if-else 양자택일
조건에 따라 **둘 중 하나만** 실행되는 구조입니다.

- **동작:** 조건이 `true`이면 A만 하고 끝납니다. 조건이 `false`이면 B만 하고 끝납니다.
- **특징:** `Code A`와 `Code B`는 절대 동시에 실행될 수 없습니다.
```
if (isRainy) {
  console.log("우산을 챙기세요 (Code A)");
} else {
  console.log("가볍게 외출하세요 (Code B)");
}
// 결과: 비가 오거나 안 오거나, 둘 중 하나의 메시지만 출력됨.
```

### if 단독구조 : 조건이 충족할때만 작업수행하고, 그 뒤는 조건과 상관없이 실행

특정 조건일 때만 **추가 작업을 수행**하고, 그 뒤의 코드는 **조건과 상관없이** 실행되는 구조입니다.

- **동작:** 조건이 `true`이면 A를 수행하고 이어서 B도 수행합니다. 조건이 `false`이면 A를 건너뛰고 바로 B만 수행합니다.
- **특징:** `Code B`는 무조건 실행되는 '공통 로직'입니다.
```js
if (hasCoupon) {
  console.log("할인을 적용합니다 (Code A)");
}
console.log("결제를 진행합니다 (Code B)");

// 결과: 
// 1. 쿠폰이 있으면: 할인 적용 -> 결제 진행
// 2. 쿠폰이 없으면: 결제 진행
```
### 💡 정리 및 팁

|**구분**|**구조**|**핵심 포인트**|
|---|---|---|
|**Case 2-1**|`if - else`|**배타적 선택**. 하나가 실행되면 다른 하나는 죽습니다.|
|**Case 2-2**|`if` 단독|**필터링/추가 작업**. 특정 조건에서만 '덤'으로 실행하고 갈 길을 갑니다.|

## 조건문 중첩 nested if
하나의 조건을 만족한 상태에서 다시 세부 조건을 따져야 할 때 사용합니다.

- **구조:** 큰 범주의 조건을 먼저 거르고, 그 안에서 상세 케이스를 나눕니다.
- **주의:** 중첩이 너무 깊어지면 코드 읽기가 힘들어지는 '콜백 지옥' 같은 상황(Pyramid of Doom)이 발생할 수 있습니다.
```js
if (isLoggedIn) { // 1차 관문: 로그인 여부
  if (isAdmin) {
    console.log("관리자 페이지로 이동합니다. (Code A)");
  } else {
    console.log("일반 사용자 페이지로 이동합니다. (Code B)");
  }
} else {
  console.log("로그인이 필요합니다.");
}
```

## 논리연산자와 조건문
여러 조건을 한 줄에서 평가하여 코드를 간결하게 만듭니다.
#### ① AND 연산자 (`&&`) - "모두 참이어야 함"

- **활용:** 모든 제약 사항을 통과해야 할 때 사용합니다.
- `if (isAdult && hasTicket)` : 성인이고 티켓도 있어야 입장 가능.
    

#### ② OR 연산자 (`||`) - "하나라도 참이면 됨"
- **활용:** 여러 대안 중 하나만 충족해도 될 때 사용합니다.
- `if (isWeekend || isHoliday)` : 주말이거나 공휴일이면 휴무.
    

#### ③ NOT 연산자 (`!`) - "결과를 뒤집음"

- **활용:** 특정 조건이 **아닐 때** 실행하고 싶거나, 복잡한 논리 결과를 반전시킬 때 사용합니다.
- `if (!isValid)` : 유효하지 않다면 에러 처리.
- `if(!(조건&&조건))` : 두조건이 유효하지 않다면 로직실행

## 중첩을 줄이는 방법 - early return 권장
중첩 조건문(nested if)은 논리 연산자를 쓰거나 **Early Return**을 사용하여 더 깔끔하게 바꿀 수 있습니다.

```js
if (user) {
  if (user.isPremium) {
    console.log("프리미엄 혜택 적용");
  }
}
```
논리연산자로 개선
```js
if (user && user.isPremium) {
  console.log("프리미엄 혜택 적용");
}
```
Early Return으로 개선 - 권장

```js
if (!user) return; // 유저가 없으면 바로 종료
if (!user.isPremium) return; // 프리미엄이 아니면 바로 종료

console.log("프리미엄 혜택 적용"); // 모든 조건 통과 시 실행
```

## switch문

 `switch` 문은 비교해야 할 대상이 명확하고 선택지가 많을 때 `if` 문보다 훨씬 더 직관적이고 깔끔한 코드를 만들어줍니다. 
 
 특히 **"특정 값에 따라 변수를 설정"**하는 용도로 아주 훌륭한 도구
```js
 switch(expression) {
 *           case choice1 :
	 *            run this code;
	 *            break;
 *            .....원하는만큼 많은 case를 추가하시면 됩니다.
 *          default:
 *            run this code;
 *        }
```
- `switch` 문은 괄호 안의 표현식(expression)을 계산한 뒤, 그 결과와 일치하는 첫 번째 `case` 절로 프로그램의 제어를 이동시킵니다.
- **case choice:** `expression`과 비교할 값입니다. (이때 비교는 엄격한 비교 `===`를 사용합니다.)
- **break:** 현재의 `case`가 끝나면 `switch` 문 전체를 빠져나가게 합니다. 만약 생략하면 다음 `case`의 코드까지 연달아 실행됩니다. (**Fall-through** 현상)
- **default:** 어떤 `case`와도 일치하지 않을 때 실행되는 '나머지' 구문입니다. (생략 가능하지만 권장됨)

### Switch 문의 특징과 장점

- **가독성:** `if...else if...else if`가 반복되는 것보다 구조가 명확해 보입니다.
- **명확한 값 비교:** 범위(예: `score > 90`)가 아닌 **특정 값**(`case 'A'`)을 비교할 때 강력합니다.
- **다중 선택:** 여러 케이스에 대해 동일한 코드를 실행하고 싶을 때 유연하게 대처할 수 있습니다.


### 실무 응용: 여러 케이스 묶기 (Grouping Cases)

`break`가 없으면 다음으로 넘어가는 특징을 역이용하여 공통 로직을 처리할 수 있습니다.

```js
let browser = 'Chrome';

switch (browser) {
  case 'Edge':
  case 'Chrome':
  case 'Firefox':
    console.log('지원되는 브라우저입니다.');
    break;
  case 'IE':
    console.log('지원이 종료된 브라우저입니다.');
    break;
  default:
    console.log('알 수 없는 브라우저입니다.');
}
```
위 로직을 보면 case1,2,3번을 묶어 공통으로 처리하고 있습니다. 

### 💡 정리 포인트

- `switch` 문 안의 `case` 값은 숫자, 문자열 뿐만 아니라 변수나 표현식도 올 수 있습니다.
    
- 반드시 `break` 처리에 신경 써야 의도치 않은 버그를 막을 수 있습니다.
    
- 복잡한 논리나 범위 계산이 필요하다면 `switch` 보다는 `if` 문을 사용하세요.
